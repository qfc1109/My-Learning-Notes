### 7.数组的排序

+ 借助于工具类Arrays

  ```java
  Arrays.sort(数组):对数组默认升序排列
  Arrays.sort(数组，比较器对象):可以定义数组规则排序顺序
  
  //有一个员工类（id,name,sal,...）的数组，根据员工的工资降序排列，如果工资相同了，再根据员工id升序排列
  Emp[] emps={
  	new Emp(10,"张三1",8000),
      new Emp(20,"张三2",8000),
      new Emp(30,"张三3",8000),
      new Emp(40,"张三4",8000),
      new Emp(50,"张三5",8000),
  };
  Arrays.sort(emps,(e1,e2)->{
      int result=e2.sal-e1.sal;
      if(result==0) result=e1.id-e2.id;
      return result;
  });
  
  ```

+ 借助于排序算法实现

  + 冒泡：两两比较排大小，第一个和第二个比，然后第二个和第三个....

    ```java
    for(int i=0;i<nums.length-1;i++){//轮数
    	for(int j=0;i<nums.length-1-i;j++){//每轮比较次数
    		if(nums[j]>nums[j+1]){//互换
                //加减法 
                //乘除法
                //异或法
                //临时变量
                nums[j]=nums[j]^nums[j+1];
                nums[j+1]=nums[j]^nums[j+1];
                nums[j]=nums[j]^nums[j+1];
            }
    	}
    }
    ```

  + 选择：第一个元素和后面元素比较，找出最小值小标，然后和最小值互换，下一轮从第二个元素开始和后面比较，找出第二小的下标，最后和第二个值互换，..

    ```java
    for(int i=0;i<nums.length-1;i++){
        int index=i;
    	for(int j=i;j<nums.length;j++){
    		if(nums[j]>nums[index]) index=j;
    	}
        if(index=!i){
            nums[i]=nums[i]+nums[index];
            nums[index]=nums[i]-nums[index];
            nums[i]=nums[i]-nums[index];
        }
    }
    ```

  + 插入：将数组分成有序部分和待插入部分，先将待插入部分和前面有序部分，从后往前依次比较，找出插入的位置，这样就会形成新的有序部分，....

    ```java
    int[] nums={10,5,7,3,4};
    //第一轮：有序部分：10，待插入部分：5
    //第二轮：有序部分：5 10，待插入部分：7
    //第三轮：；有序部分：5 7 10 待插入部分：3
    //...
    for(int i=0;i<nums.length-1;i++){
        for(int j=i+1;j>0;j--){
            if(nums[j]>nums[j-1]){
                nums[j]=nums[j]*nums[j-1];
                nums[j-1]=nums[j]/nums[j-1];
                nums[j]=nums[j]/nums[j-1]
            }
        }
    }
    ```

  + 快速

    + 首先：设置一个基准数，假设拿第一个值做基准数
    + 然后：设置两个下标，一个是开始下标begin，一个是结束下表end
    + 然后：先从后去扫描，和基准数比较大小
      + 如果比基准数大，要继续和前一个在比 end-1
      + 如果比基准数小，让end下标的值给begin下标的值赋值
    + 然后：再从前去扫描，和基准数比较大小
      + 如果比基准数大，让begin下标的值给end下标的值赋值
      + 如果比基准数小，要继续比较后一个让begin+1
    + 进行了很多次的比较end-1，begin+1 如果发现end==begin，说明他就是基准数的正确下标
    + 最后通过递归，把基准数前面的和后面的元素，再重新来一遍这种规则的排序，整个数组就有顺序了

+ 二分查找：在一个有序数组的基础上，进行折半查找，就是每次查询中间值

  ```java
  public int find(int[] nums,int n){
  	int begin=0;
      int end=nums.length-1;
      while(begin<=end){
          int might=(begin+end)/2;
          if(nums[mid]>n){
              end=mid-1;
          }else if(nums[mid]<n){
              begin=mid+1;
          }else{
              return mid;
          }
      }
      return -1;
  }
  ```

| 排序算法 | 平均时间复杂度 | 空间复杂度 | 稳定性 |
| -------- | -------------- | ---------- | ------ |
| 冒泡     | O(n^2)         | O(1)       | 稳定   |
| 选择     | O(n^2)         | O(1)       | 不稳定 |
| 插入     | O(n^2)         | O(1)       | 稳定   |
| 快速     | O(nlogn)       | O(nlogn)   | 不稳定 |
| 二分     | O(nlogn)       | O(1)       |        |

#### 9.6 overload（重载）和override（重写）的区别

+ overload：一般发生一个类中，要求方法名一样，参数不同（个数，顺序，类型不同），跟返回值无关

  ```java
  class B{
      public void test (String a){}//属于重载
  }
  
  class A extends B{
  	public void test(){}
      public 无所谓 test(int a){}//属于重载
      public void test(int b){}//报错了，不属于重载
      //构造方法重载
      public A(){}
      public A(int a){}
  }
  
  new A().test();
  new A().test(String);
  new A().test(int);
  ```

+ override：一般发生在具有父子关系的两个类中

  + 访问修饰符：一样或者不能严于父类（父类修饰符 protected 子类可以写protected，也可以public ，不能写private...）
  + 返回值类型：相同或者可以是其返回值的子类（父类返回值是object类型，子类写什么类型都可以）
  + 方法名：相同
  + 参数：相同
  + 异常：一样或者不能比父类抛出更多的异常

#### 9.8 static

+ static用法：
  + 修饰属性：静态属性（类属性），属于整个类的，所有对象共享
  + 修饰方法：表示静态方法（类方法），静态方法只能调用静态属性和静态方法，但是普通方法可以随意调用静态方法的
  + 修饰代码块：表示静态代码块，类加载时最先执行一次
+ 为什么要用static（好处）：
  + 共享：同一个类所有的对象可以共享同一个静态变量，一个对象修改了，其他对象使用时也会修改
  + 使用方式：可以通过对象名来调用，也可以通过类名调用，可以不依赖于对象，推荐使用类名.静态变量()
  + 内存：成员变量存储在堆内存中，静态变量存储在方法区，这样对象创建越多，成员变量也会需要更多的内存，但是静态变量的位置没有变，不会浪费额外内存
+ 静态变量和成员变量区别：
  + 生命周期：
    + 静态变量是属于类的，所有对象共享，随着类加载而初始化
    + 成员变量是每个对象独有的，会随着对象的创建和回收，也会跟着创建和回收，创建对象时初始化
  + 存储位置：成员存储在堆内存中，静态变量存储在方法区中
  + 使用方式：成员变量只能通过对象名访问，静态变量还可以通过类名访问



#### 9.9 final修饰符

+ 修饰类：表示最终类，不能被其它类继承，比如：String类就不能有子类
+ 修饰属性：表示最终属性（常量），表示一旦复制，不能再次修改，比如：Math.PI
+ 修饰方法：表示最终方法，不能被其它类重写，但是可以重载

### 10 抽象类和接口的区别

### 11.String

String不是基本数据类型，他是特殊的引用类型，底层是通过final修饰的char[]，表示一个常量，表示String一旦赋值，不能修改，每次修改会在常量池创建新的常量，所以就不适合做大量字符串修改



#### 11.1 new String("java")创建几个对象

答案：1个或者2个，new String()在堆内存中创建一个对象，"java"表示在常量池中的char[]由于常量池的常量不能重复，如果常量池已经存在这个值，就不会再创建新的了，反之，没有的话才会创建新的常量



#### 11.2 ==和equals区别

+ == ：如果是基本类型，用来比较值是否一致，如果是引用类型，用来比较地址是否一样
+ equals：是属于Object类的方法，底层实现默认是和==一致的，所以如果没有重写equals比较规则和前面一致，那么重写了，就可以按照自己重写后的规则来比较，比如：Integer类，String类都重写了equals
  + String类重写规则：先判断==比较地址是否一致，如果一致直接返回true，如果不一致，在判断参数类型是否是String，如果不是字符串返回false，再判断长度是否一致，如果不同返回false，最后在比较字符串中的每个字符是否一直





#### 11.3 String和StringBuilder和StringBuffer的区别

StringBuffer和StringBuilder底层不是通过final修饰的char[]不是常量，可以修改，所以每次对字符串修改，不会创建新的值，所以大量字符串修改时，效率比String更好

StringBuffer是线程安全的，执行效率低，因为操作字符串时，其他用户无法操作需要等待操作结束释放锁，才可以操作

StringBuilder，是线程不安全的，执行效率高，因为操作字符串时，其他用户也可以操作

#### 12.3 Integer a=130;Integer b=130; 如果a==b什么结果

+ 异常：程序在运行过程中发生了一些不正常事件，他会让程序的执行没有按照开发者的预期，如果异常发生了没有解决，还会中断程序

+ 错误：是系统内部运行错误，不一定是程序造成的，由于系统内部资源耗尽出现的问题，并不能通过程序解决，java异常机制不用处理这些问题，比如：内存溢出，内存泄漏

  + 栈内存溢出：通过递归，没有提供出口，就会无限递归

    ```java
     public static int testStackOverFlow(int n) {
            return n * testStackOverFlow(n - 1);
     }
    ```

    

  + 堆内存溢出：

    ```java
     public static void testHeapOverFlow() {
            List<User> list = new ArrayList<>();
            while (true) {
                //list也会无限扩容，user也会无限new，不会回收

            }
    ```

#### 13.3 运行时异常和检查性异常的区别，常见的异常

- 运行时：继承于RuntimeException，编译不会出错，是程序在运行时有可能出现错误，是由于编译不合理造成的，是可以避免的，所以java异常机制是可以处理也可以不处理
  - 空指针异常：null.属性，null.方法()
  - 数组越界异常：数组[-1]
  - 字符串越界异常：charAt(-1)
  - 类型转换异常：比如：Pet p=new Dog();Cat c=(Cat) p;
  - 参数输入异常：Scanner通过nextInt接受整数，如果输入字符串
  - ...
- 检查性：继承于Exception，编译不会出错，如果不处理程序无法执行，所以java异常机制，必须要处理
  - 日期格式化异常：String转换日期时，内容和日期格式不匹配
  - 线程中断异常：调用sleep()，wait()，join()...发生的
  - IO异常：通过IO流对文件进行读写发生的
  - 文件丢失异常：文件地址不存在发生的
  - SQL异常：执行数组库操作的时候，比如：sql语句写错了
  - ...



#### 13.4 异常机制处理方式

+ 抛出异常：关键字throws，写在方法声明的位置，用于把异常抛给上一级，自己不处理

  ```java
  public static void main(String[] args) throws IOException{
  	test();
  }
  
  public static void test() throws IOException{
  
  }
  ```

+ 捕获异常：关键字try-catch-finally，只有这种方案才是真正解决异常

  ```java
  try{//尝试，把可能出现异常的代码，放进来，如果这些代码出现了异常才会被处理，如果放在try外面的代码出现异常，不会处理
      
  }catch(异常类型 异常名称){//捕获，try里面发生的异常，会和catch里面声明的异常进行匹配，如果匹配上了才会处理，否则依然报错
      //可以写多个catch，从小范围到大范围编写
  
  }catch(){
      
  }finally{//总是执行，无论是否发生异常都会执行
  	
  }
  ```

  

+ 自定义异常：关键字 throw 一般写在方法里面的，用于声明一个新的异常，还可以根据用户需要，来设置自己的异常规则，比如：控制身高，性别，年龄...

  + 自定义运行时异常：首先继承RuntimeException然后添加有参构造并调用父类的有参构造

    ```java
    class SexException extends RuntimeException{
    	public SexException(String msg){
            super(msg);
        }
    }
    ```

  + 自定义检查性异常：继承Exception，添加有参构造，并且调用父类有参构造



#### 13.5 try-catch-finally 如果有return 会如何执行

答案：try和catch执行哪个return要看是否发生异常，发生异常了返回catch的return，反之返回try的return，最后再看finally是否有return，如果有return，则会替换前面try和catch要返回的结果，如果没有return，则就返回try或catch里面的return



### 14.Object

Object类是java.lang包的资源，无需导入包就可以直接使用的，是java中所有类的父类，提供了很多公开的方法，这样任何类都会具有这么多方法



#### 14.1 Object有哪些常用方法

+ equals：等价于==比较地址是否一样
+ toString：默认会返回对象的引用地址，如果重写了，打印对象时就是你重写的结果
+ hashcode：用于表示对象基本特征，一般是由32位二进制数构成，所以结果是int
  + 第一类：对象类，根据对象引用地址计算得到的
  + 第二类：基本类型封装类，保存的就是对应的数值
  + 第三类：String类，根据字符串内容计算得到的
+ getClass：反射的前提，获取类对象
+ wait：让线程进入等待，可以释放对象锁，需要其他线程来唤醒才会执行
+ notify：唤醒随机一个处于等待的线程
+ notifyAll：唤醒全部处于等待的线程
+ clone：克隆对象
+ finalize：java中gc垃圾回收的方法，java会自动调用，防止内存溢出



#### 14.2 java几种创建对象的方式

+ new对象：底层需要调用构造方法
+ 反射：通过newInstance，底层也会调用构造方法

#### 14.3 深克隆和浅克隆

+ 浅克隆：只能克隆类中基本类型和String类型的属性，其他的引用类型不会克隆，只是把地址复制过来（克隆前和克隆后，引用类型是同一个）

  + 如何实现浅克隆：首先先实现Cloneable接口，表示支持克隆，然后再重写Object类的clone()

    ```java
    public class Demo3 {
        public static void main(String[] args) throws CloneNotSupportedException {
            One o = new One();
            o.two=new Two();
            o.id = 10;
            o.name = "jj";
            o.two.age = 18;
            o.two.sex = "男";
            One o2 = (One) o.clone();
            System.out.println(o);
            System.out.println(o2);
            o.two.age=100;
            System.out.println(o.two);
            System.out.println(o2.two);
            System.out.println(o2.two.age);
        }
    }
    
    class One implements Cloneable {
        Integer id;
        String name;
        Two two;
    
        protected Object clone() throws CloneNotSupportedException {
            return super.clone();
        }
    }
    
    class Two {
        String sex;
        Integer age;
    }
    ```

    

+ 深克隆：无论对象中的基本类型和String类型，它的引用类型的属性也可以克隆

  + 如何实现深克隆：首先先实现Cloneable接口，表示支持克隆，然后再重写Object类的clone()，再然后如果有引用类型属性，也需要实现Cloneable接口，重写clone()，并且clone方法内部还需要调用引用类型的属性的clone()；

    ```java
    public class Demo3 {
        public static void main(String[] args) throws CloneNotSupportedException {
            One o = new One();
            o.two = new Two();
            o.two.three = new Three();
            o.id = 10;
            o.name = "jj";
            o.two.age = 18;
            o.two.sex = "男";
            One o2 = (One) o.clone();
            System.out.println(o);
            System.out.println(o2);
            o.two.age = 100;
            System.out.println(o.two);
            System.out.println(o2.two);
            System.out.println(o.two.age);
            System.out.println(o2.two.age);
            System.out.println(o.two.three);
            System.out.println(o2.two.three);
        }
    }
    
    class One implements Cloneable {
        Integer id;
        String name;
        Two two;
    
        protected Object clone() throws CloneNotSupportedException {
            One one = (One) super.clone();
            one.two = (Two) one.two.clone();
            return one;
        }
    }
    
    class Two implements Cloneable {
        String sex;
        Integer age;
        Three three;
    
        protected Object clone() throws CloneNotSupportedException {
            Two two = (Two) super.clone();
            two.three = (Three) two.three.clone();
            return two;
        }
    }
    
    class Three implements Cloneable {
        Integer id;
        String name;
    
        protected Object clone() throws CloneNotSupportedException {
            return super.clone();
        }
    }
    ```

    



### 15.集合分类

+ Collection：集合父接口
  + List集合接口：
  + Set集合接口：

+ Collections：类似于Arrays是集合工具类，封装了很多集合处理好的方法



#### 15.1 list和set和map区别

+ list：元素是有序的，可重复的
+ set：元素是无序的，不可重复的（唯一的）
+ map：基于key和value存储的，key是唯一的，value是可以重复的



#### 15.2 ArrayList和LinkedList区别

+ ArrayList：数组实现，查询比较快，增删比较慢，默认长度10，第一次add的时候才会初始化10，扩容1.5倍

  > 面试题：List list=new ArrayList(20);扩容多少次？
  >
  > 答案：0次，因为走的是有参构造，参数就是默认长度

+ LinkedList：链表实现，查询比较慢，增删比较快，理论上链表没有长度限制，只要内存无限大，可以无限存储



#### 15.3 Set集合实现类

+ hashSet：底层就是通过HashMap实现，存储的每个元素，其实就是存储到map中作为key值，可以保证数据是唯一的没有顺序的
+ TreeSet：底层是通过TreeMap实现，默认支持自然排序（数字默认升序，String是按位比较ascii），如果存储的是对象类型，默认不能排序，因为不知道按照对象哪个属性排序，所以需要添加比较器对象（Comparator）来实现比较规则



#### 15.4 HashSet或者HashMap底层原理

底层是通过哈希表实现的，包括数组+链表+jdk1.8之后红黑树构成的，默认长度16，负载因子0.75，达到长度的75%，开始扩容2倍

+ 存储元素的（put）原理：先根据key调用hashcode()计算哈希值，再通过位运算计算数组下标，再通过==判断是否是同一个对象，再然后调用equals()判断内容是否一致，如果内容相同，则需要替换，如果内容不同，就属于hash冲突，就会通过这个数组节点指向下一个节点，形成一个链表（解决hash冲突），如果链表长度超过8，并且数组元素超过64，会影响性能，才会将链表转换成红黑树，红黑树节点个数小于6，转换回链表
