# 学习文档模板

用途：以后需要为某个工作副本、某批笔记或某个学习主题生成复习计划时，按这个模板输出。  
核心风格：模块化、可勾选、每个知识点带索引、方便让 Codex 继续出题和批改。

## 生成规范

- 文档不要强绑定固定日期，除非用户明确要求日期计划。
- 使用 `- [ ]` 做可勾选清单，复习完一个知识点就能直接勾选。
- 按知识模块分组，例如 Java 基础、集合、多线程、数据库、框架、中间件、部署、面试。
- 每个模块先写“相关文档”，再列知识点。
- 每个知识点后面都写“索引”，尽量链接到具体标题位置。
- 如果找不到完全对应的标题，链接到最接近的上级标题。
- 如果只能定位到文档，先补文档级索引，但后续应尽量升级为标题锚点。
- 复习规则、错题占位、已完成模块这类非知识点清单可以不写索引。
- 保留“Codex 出题提示词”，方便用户后续让 Codex 生成复习题。
- 保留“错题与薄弱点”和“已完成模块”，方便持续维护。

## 索引规范

优先级：

1. 精确标题：知识点能匹配到文档标题时，链接到该标题。
2. 近似标题：知识点是标题的一部分，或标题是知识点的一部分，链接到该标题。
3. 上级标题：知识点在正文里出现但不是标题时，链接到最近的章节标题。
4. 文档级索引：确实找不到标题时，链接到最相关文档。

Markdown 写法：

```md
- [ ] 知识点名称 （索引：[文档.md § 标题](./文档.md#标题锚点)）
```

示例：

```md
- [ ] Java 为什么能跨平台运行 （索引：[java笔记/1020.md § java为什么能跨平台运行](./java笔记/1020.md#java为什么能跨平台运行)）
```

文件名包含括号时，用尖括号包住链接：

```md
- [ ] Git 基础 （索引：[Git(版本控制).md § 3.git](<./Git(版本控制).md#3git>)）
```

## 生成流程

- [ ] 扫描当前工作副本中的 Markdown 文档。
- [ ] 提取文档标题和目录结构。
- [ ] 根据文档主题划分复习模块。
- [ ] 从标题和正文中整理知识点。
- [ ] 为每个知识点建立索引。
- [ ] 检查本地文件链接是否存在。
- [ ] 检查标题锚点是否能匹配源文档标题。
- [ ] 输出最终学习文档。

## Codex 生成提示词

```text
请按当前仓库中的《学习文档模板.md》风格，为我生成一份学习文档：
1. 不要按固定日期安排，按知识模块分组
2. 每个知识点都用 - [ ] 可勾选格式
3. 每个知识点都建立索引，尽量链接到具体标题位置
4. 每个模块写清楚相关文档
5. 保留 Codex 出题提示词、错题与薄弱点、已完成模块
6. 生成后检查本地链接和标题锚点是否有效
```

## 空白模板

# 《学习主题》复习清单

生成日期：YYYY-MM-DD  
使用方式：不用按固定日期走，复习完一个知识点就在前面打勾。需要练习题时，可以直接把对应模块或文档名发给 Codex，让 Codex 根据笔记生成复习题、填空题、选择题、代码题或面试追问。

## 复习规则

- [ ] 每次复习先选一个模块，不要同时开太多主题。
- [ ] 每个知识点至少做到：能说清楚概念、能写出关键代码或流程、能回答 1 到 3 个面试追问。
- [ ] 标有“重点、难点、面试题、高频面试题、必考题”的内容优先复习。
- [ ] 复习完一个模块后，让 Codex 根据相关文档生成一组复习题。
- [ ] 做错或答不完整的题，记录到“错题与薄弱点”。

## Codex 出题提示词

```text
请根据我当前工作副本中的《文档名》给我生成复习题：
1. 先出 10 道基础概念题
2. 再出 5 道面试追问题
3. 再出 3 道代码或流程题
4. 等我回答后再批改，并指出我应该回看文档里的哪些知识点
```
