## java为什么能跨平台运行

1. java程序是（一次编译，跨平台运行的）跨平台运行靠的是jvm（java虚拟机）

   1. `jvm的作用，把java字节码（.class文件）编译成当前操作系统能理解的机器码`

      不同的操作系统有各自对应的jvm，但都遵守同一套规范

2. 详细的说一下

   第一步：`编译过程`

​			通过javac命令把java源码（.java文件）编译成字节码（.class文件)

​          第二步：`解释过程`

​          通过java命令读取字节码文件（.class文件）

3. 跨平台的优缺点

​			优点：开发效率高，不用为不同系统写多套代码

​			缺点：需要安装JVM

​							性能比直接编译成机器码的编程语言略低



## java代码是如何运行的

第一步：通过javac命令将java源代码编译成字节码(.class文件)

第二部：类加载器将字节码加载到jvm里

第三步：通过java命令逐条翻译字节码，并运行

 过程：编译➡加载➡运行

## JVM, JRE ,JDK分别是什么

#### JVM(java虚拟机)

将.class字节码文件翻译成机器能懂的0101指令，所以能一次编译，到处执行，相同的字节码在不同的操作系统的JVM上都能运行



#### JRE(java运行环境)

运行java程序的工具包，包含JVM+核心类库（java.lang包）



#### JDK(JAVA开发工具包)

包含JRE+编译工具（javac）等



## java的数据类型

### 整数类型

byte 占1字节，处理数值小时用，I/O流中处理数据流

short 占2字节

int 占4字节，默认的整数类型

long 占8字节，处理数值大时用

### 浮点类型

float 占4字节，带f后缀

double 占8字节，默认的浮点类型

### 字符型

char 占2字节，存Unicode字符	

布尔型

boolean 占1字节，逻辑判断专用，只有真假



### char 为什么能够存储汉字？

因为Java支持Unicode编码，char固定2字节，刚好对应utf-16编码的基本单元



## &&和&的区别

### &&（短路与）

当前一个条件为false时,直接跳过后面判断，不会空指针



### &（非短路与）

不管前面条件如何，后面条件一定执行

&还用作位运算操作



总结：

逻辑判断优先用&&，可以避免空指针和无效运算，提高效率

需要强制执行所有条件时，用&（比如资源释放，日志记录等）

位运算场景用&
