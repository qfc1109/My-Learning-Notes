### 6.hashMap底层原理

先给一个初始化的数组，默认值16，16个元素

为了找下标更快速，用hash算法，去找

数组中的每个元素是一个链表，有很多节点，默认给8个

String[] arr=new String[10]

arr[0]---LinkedList--八个节点

```
Node<K,V>[] arr=new Node<K,V>[8]
Node[] arr=new Node[8]
```

当本链表的节点数大于8时，并整个数组元素大于64,又把对应的链表转化成红黑树，

树的查找  插入 删除 等，效率比链表还要高，

整个思路：

用到数组 链表 ，本链表的节点数大于8时，并整个数组元素大于64,又把对应的链表转化成红黑树，

看源码：

```
public V put(K key, V value) {//1 "aa"
    return putVal(hash(key), key, value, false, true);
}
```

hash(key) 通过hash算法去找

```
static final int hash(Object key) {
    int h;
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}
```

^ 异或运算 

0011

1011   相同赋0，不同的赋1     --》1000 得到的值

hash(Object key) 找值快

```
final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
               boolean evict) {
               //tab是个数组 p是某个节点
    Node<K,V>[] tab; Node<K,V> p; int n, i;
    //一开始是整个数组是空的，
    if ((tab = table) == null || (n = tab.length) == 0)
        n = (tab = resize()).length;//resize 扩容 给的初始值是16
    if ((p = tab[i = (n - 1) & hash]) == null) //如果数组的元素是空的，
        tab[i] = newNode(hash, key, value, null);//就给一新节点
    else {//说明数组中本元素是有值
        Node<K,V> e; K k;
        if (p.hash == hash &&  //如果key的值相同，就覆盖
            ((k = p.key) == key || (key != null && key.equals(k))))
            e = p;
        else if (p instanceof TreeNode)///如果key不相同,已转化成树了，按树的方式的增加值
            e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
        else {//key不同
            for (int binCount = 0; ; ++binCount) {
                if ((e = p.next) == null) {
                    p.next = newNode(hash, key, value, null);//新来一个Node给next
                    //当节点数大于8，就转化成红黑树
                    if (binCount >= TREEIFY_THRESHOLD - 1) // -1 for 1st
                        treeifyBin(tab, hash);
                    break;
                }
                if (e.hash == hash &&
                    ((k = e.key) == key || (key != null && key.equals(k))))
                    break;
                p = e;
            }
        }
        if (e != null) { // existing mapping for key
            V oldValue = e.value;
            if (!onlyIfAbsent || oldValue == null)
                e.value = value;
            afterNodeAccess(e);
            return oldValue;
        }
    }
    ++modCount;
    if (++size > threshold)
        resize(); //扩容
    afterNodeInsertion(evict);
    return null;
}
```

```
V value;
Node<K,V> next;  节点，只有二个值，一个是本身的值，指向下一个
```

- 它内部是一个 `Node[]` 数组，每个位置为一个元素，叫一个“桶”，每个桶里可能是链表或者红黑树。
- 插入元素时先算 key 的 hash 值，找到数组下标，如果冲突了就用链表串起来。
- 当链表长度超过 8，并且数组长度大于等于 64 的时候，链表才会转成红黑树，这样查找效率从 O(n) 提升到 O(log n)。
- 默认初始容量是 **16**，数组元素个数16，负载因子是 **0.75**。也就是说当元素数量超过 16 × 0.75 = 12 的时候，就会扩容，变成原来的 **2倍**。
