### 10.线程池中七个参数

1.corePoolSize 线程池核心线程数,有执行任务的线程数

2.maximumPoolSize 可纳的最大线程数 此值要>=1

3.keepAliveTime  空闲线程的存活时间，比如 线程池中有10个线程，

有的在执行，有的是空闲，当存活时间达到这个值时，空闲的线程就会销毁。

4.TimeUnit 时间单位 ，可以s 秒或  m分

5.workQueue:任务队列，通常指没有提交执行的任务的线程就放到此队列，在等待。也叫阻塞队列

6.threadFactory:线程工厂，生成线程池用的。默认就可以

7.handler  拒绝策略, 当 当前线程队列满了,还有线程过来，

要考虑如何拒绝请求的方式

### 11.拒绝策略分四种：

1.CallerRunpolicy: 依然会直接执行任务run,当线程池shutdown,不执行任务。抛弃任务

2.abortPolicy 系统默认的

直接抛弃任务 并给个异常

3.discardPolicy  直接抛弃任务，什么都不做

4.discardOldestPolicy  抛弃最早进入队列的那个任务,尝试把这次要拒绝的任务进入等待队列。

### 12.阻塞队列

是一个队列，先进先出的原则，当队列已满，还在再进，要阻塞，

当队列是空时，还要出，

ArrayBlockingQueue: 基于数组的有界阻塞队列

案例：

比如 有三个元素，阻塞队列中，只放二个元素，

只有二个，第三个是不会进的。当操作第三个时，会抛出异常

LinkedBlockingQueue:基于链表的无界的阻塞队列

存一个，取三次，也不会报错

SynchronousQueue:没有存储元素的阻塞队列

都实现了BlockingQueue接口

### 13.自定义线程池

根据七个参数，自己定义一个线程池

```
package com.day1124;
import java.util.concurrent.LinkedBlockingQueue;
import java.util.concurrent.ThreadPoolExecutor;
import java.util.concurrent.TimeUnit;
public class PoolDemo {
    public static void main(String[] args) {
        int corePoolSize=2;
        int maximumPoolSize=4;
        int keepAliveTime=10;
        TimeUnit unit = TimeUnit.SECONDS;//单位 秒
        //任务队列中，类型是些Runnable的类型
        LinkedBlockingQueue<Runnable> workQueue=
                new LinkedBlockingQueue<>();
        ThreadPoolExecutor te=new
                ThreadPoolExecutor(corePoolSize, maximumPoolSize,
                keepAliveTime, unit, workQueue);

        for(int i=0;i<5;i++){
            Runb b=new Runb();
            te.submit(b);
        }
        //for结束
        te.shutdown();
    }

}
class Runb implements Runnable{
    public void run(){
        System.out.println(Thread.currentThread().getName()+"start");
        System.out.println(Thread.currentThread().getName()+"end");
    }
}
```

### 14.指令重排

jvm 为了提升性能，对字节码进行指令重排，以优先性能，以满足硬件的要求。

重排：重新排列顺序

为什么改变顺序，能获得性能优化

改变操作顺序，减少内存的无效访问，以提升命中率

问题：cpu一直不停在运行，有序性会有问题，如果遇到多线程操作同一个数时，数值会有问题。

多个线程操作x变量,导致x的值会有问题

原因：

先看y,当线程1 a=1;  线程2，y=a; 线程2没有读到线程中的a=1

### 15.规避指令重排

1.volatile关键字

可见性：指一号线程修改了共享变量的值，其它线程能得到这个修改，修改对自己和别的线程是可见的。

使用方法，加到变量前

2.原因：

没加volatile之前，cpu在执行中，会把数据加载到缓存中，

不在主存中，操作时，先操作缓存中，没有把缓存中的数据时时的更新到主存中。

加上volatile之后，每操作完一次变量，都会把检查主存中是不是最新的结果或是不是有更新。

如何检查：

只是cpu有内存数据更新，cpu检查内存数据时，创建一个副本，

比如此副本数据与主存数据，没有把此副本复制给所有的线程

### 16.volatile只能保证可见性，不能保证原子性

原子：物质的最小单元，不可再分割。

因为有这个特性，计算机在执行原子操作，一般会遵守

要么就操作完，要么不做。

比如，0101011 取反 不可能只操作一部分，

案例：来多个线程，每个线程对count增加1000，100个线程

总和100000
