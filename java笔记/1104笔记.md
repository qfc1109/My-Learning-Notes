### 4.抽象工厂：

抽象工厂，就是在简单工厂之上，把工厂类中的创建对象的方法，做了一个分离。

```
class Fact{
    //返回一个Message类型
    public Message pro(String type){
        //
        if(type.equals("v")){
            return new VXMessage();//如果传入v,就创建VXMessage
        }else if(type.equals("e")){
            return new EmailMessage();
        }else{
            System.out.println("要输入正确的值");
            return null;
        }
    }
}
```

6.1 error是错误，导致程序不能正常运行

编译错误：int i; System.out.println(i);

运行时错误：值，或下标

逻辑错误：逻辑有漏洞，程序没有近期期望的效果，按正常逻辑运行。

错误一般是要通过程序员，去调试避免

错误是程序无法正常向下运行，异常本身的风险，可以通过处理，接着向下正常运行

6.2 异常：Exception

异常是程序执行的一种风险，如果处理正常，程序是可以正常运行的。

如果有异常，不正常处理，那程序就是不可运行的。

6.3 运行时异常，不检查异常：只有程序运行了，才有可能发生的异常

ArrayIndexOfoutBoundsException 数组下标越界
ArithmeticException  算术异常
NullPointerException  空指针异常

InputMismatchException 输入类型不同
StringIndexOutOfBoundsException 字符串下标越界
NegativeArraySizeException 数组长度有负数异常

IIIegalargumentException 参数不对

6.4 非运行时异常，编译异常，检查性异常，就是代码在运行之前，就已经异常了。

CheckedException 检查性异常
SqlException 数据库异常
IoException  io异常
ClassNotFoundException 类找不到异常
ParseException 格式异常

### 9.在继承中或重写异常的使用

原理：

异常的原因是那个方法的方法体有风险，

当子类重写这个方法时，要看方法体，如果子类方法体没有风险 ，可以不用抛出异常

```
ArrayIndexOutOfBoundsException数组下标越界异常
ClassNotFoundException 找不到类的异常
NegativeArraySizeException 数组长度异常
ArithmeticException 算法异常
InputMismatchException 输入匹配异常，主要是输入类型的问题
ParseException 输入的内容或类型可能是对的，只是格式没有按要求
```

运行时异常，优先使用try catch

编程思想：

1.把主要代码放到try内，catch中放对应的异常，finally放最终要走的程序

如果没有特别的是要说明要最终执行的，就不用finally

在本程序中，可以让用户二次，第一次错了，第二回是对的，就是正确的
