背的题：

1 谈谈你对Integer常量池的理解
	integer常量池机制主要是针对在-128到127的整数，在这个范围的，integer对象是共享的，当你创建一个在这个范围内的integer对象时，java会从缓存池中返回已有对象；当超出这个范围时，integer类会创建一个新对象

2.字符串常量池了解吗

jdk8之后主要有三个变化：

1.搬家了，从永久代到元空间

- 在jdk7及之前，字符串常量池是在永久代中，默认最大容量很小

- jdk8之后，搬到了堆内存中，跟着堆的空间走，不容易OOM

2.GC策略更智能

堆内存中的常量池能被正常gc

当字符串不再被引用时，会像普通对象一样被回收

元空间的类信息和字符串池分离，互不影响

字面量vs构造器

即使内容相同，new String()会创建一个新对象，但字面量只有一份

3.元空间参数调优，

可以通过调整jvm参数,查看元空间的字符串的存储和占用情况，然后进行调优



3.这句代码创建了几个对象? String str1 = new String("xyz")

分两种情况：

​		如果是首次执行，当字符串池中没有“xyz”时，字符串池和堆中各会创建一个新对象，总共创建两个

​		如果字符串池中有“xyz”，那么只会在堆内存中创建一个新对象
