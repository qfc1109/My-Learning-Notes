### 6.抽象类

有了继承关系，有父类声明一些方法，在每个子类体现出了不一定的形态

```
public void eat(){
    System.out.println("人吃饭");
}
```

在教师那，输出的效果，教师在学校吃饭

在医生那，输出的效果    医生在医院吃饭

在父类那个层面，声明方法，不能写的那么死，直接写个方法名，不写方法体,

因为每个子类他可能会重写此方法

```
public abstract void eat();
```

1 抽象方法：只有方法的声明，可以含参数，可以含返回，就是没有方法体

abstract 抽象

如果一个类中可以含或含了抽象方法，此类为抽象类

2` 抽象类：是能力可以含有抽象方法的类`，类前加abstract

什么时间须定义类为抽象类，关键看，有没有可能含有抽象方法

什么情况下，看要不要定抽象方法：关键看各个子类的方法有没有可能体现各种形态

3 多态：运行下，有子类把父类的重写了，体现了不同的形态

动物：--父类

属性  name  type 

方法  移动

鸟    name  type   飞行

鱼   name type   游动

4.抽象类能不能实例化

想：

实例化的目的是调用

Book b1=new Book();

b1.read();  能调用时，方法体是完整的

抽象类不能实例化：因为抽象类有可能会有抽象方法

5.抽象类，通过子类继承，再实例化子类发生效果的

Animal a=new Fish(); 把子类对象给父类，多态的一种表现

a.move();//传入的是fish,调用fish中的move

Animal a2=new Dog(); 把子类对象给父类，多态的一种表现

a2.move();//传入的是Dog,调用Dog中的move
