### 11.序列化

序列化：指一个对象转化成二进制流，在网络平台中传输（与txt转化），并与平台无关

把对象数据，通过io来读和存

好处：

1.可以把众多属性信息放到对象中，一并存储，省的一个一个存和取

比如

商品 id name pri addr....

2.实现了对象信息的持久化，可以永久保存在计算机的硬盘中

3.可以进入网络传输

开发过程中

1.要保证序列化时，对象信息的唯一性

2.接口  serializable    序列化接口，告诉你实现我这个接口的类信息，是要唯一

- ```
  public interface Serializable
  ```

  类的序列化由实现java.io.Serializable接口的类启用

没有提供要实现的方法

3.设置一个序列化编号

public/private static final long serialVersionUID=-12xx92L;

一般是以负值开始取值，极大的可能性保证了唯一性，

系统给的默认值是正数

4.具体序列化过程：序列化操作系统会把当前类的定义的serialVersionUID

写入保存的文件中，下回当反序化(把txt中的对象取出来),先检测serialVersionUID

判断uid是不是一致，如果是一致，可以成功取出，否则，取不出来

### 12.案例

把一个Person的对象信息，存入txt,并要成功取出来

还要得到对象相应的属性信息。

1.按正常的流程，写入uid，看不能序列化和反序列化成功 yes

2.不用uid,看不能序列化和反序列化成功  也可以成功的

3.serialVersionUID可以省去不写，系统会默认生成一个，尽量自己写，默认生成的很容易重的

一致性。

4.存时，用uid,取时，不用UID，

一致性：

1 存时用了uid,取时也要用uid

2 存时不用uid,取时也不用uid,系统会默认生成一个

3 存时和取时，uid不一致，取不出对象
