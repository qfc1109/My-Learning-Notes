### 8.同步

会有多个线程并发时出现资源占用的情况，会出现同步

同步场景：a线程请求某个资源时，此资源正在被b线程调用，如果此时启用同步，a是请求不到，怎么办，a 只能等待b执行完， a才有机会。

概念：当一个线程对某个内存资源操作时，直到此线程操作完成，其它线程才可以操作此资源

案例 ATM机，打电话

### 9.异步

概念，当一个线程对某个内存资源操作时，如果另一个线程正在操作此资源，a也是有可能操作此资源。不须等b执行完

cpu执行资源任务，是片段化执行

场景：一个考生，写一张试卷，老师批改作业，改完这个改一个，

这个改部分，改另一个人的，或过会改，

1 同步：一个线程的请求----等待处理完成---其它线程才可以请求

有这个synchronized关键字

异步：多个线程发出请求-----不用等别的线程处理-----cpu有空就会处理

没有关键字，默认就是异步

2 同步更安全，异步不安全，异步效率会更高一些

3 synchronized放到方法前

(每个对象默认是有自己的内置锁)

案例：有多个窗口卖票(1--100号),可能99号票，1 2二个窗口同时在卖

### 11.synchronized修饰代码块

代码块，也可以依赖于Object对象

对象锁 synchronized (ob)

```
synchronized(ob) {
    if(ti>0){
        System.out.println(Thread.currentThread().getName()
                +"窗口在卖票"+ti);
        try {
            Thread.sleep(10);
            System.out.println(Thread.currentThread().getName()
                    +"窗口在收费"+ti);
            ti--;
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }
    }
}
```

是如何做到的：

所有的对象都会有唯一的一个锁，jvm跟踪时，先判断有没有锁，

如果要解锁，jvm会计数 变成0（执行完），

给此对象上锁，计为1.jmv开始执行

package com.day1120;

public class WaitDemo {

            public void run(){//线程须要做的事

### 15.各种情况

1 当一个线程进入一个对象的一个synchronized方法后，其它线程能否进入此对象的其它的实例方法

可以，synchronized修饰此方法时，锁到的是此方法代码块，和其它无关。

2 当一个线程进入一个对象的一个synchronized方法后，

其它线程能否进入此对象的刚那个synchronized方法

不行，因为要抢锁

3 当一个线程进入一个对象的一个synchronized方法后，能不能此对象的其它的synchronized方法

可以的，只是要抢锁。

4 当一个线程进入一个对象的一个synchronized方法后,其它线程能否进入此对象的其它静态方法

可以的，并还是静态方法先执行，

5 当一个线程进入一个对象的一个 静态的synchronized方法后,

其它线程能否进入此对象的其它 静态 synchronized方法

不行，类的锁，控制所有的对象的锁

6  当一个线程进入一个对象的一个synchronized static方法后

其它线程能否进入此对象的其它的static 方法

可以的

总结：

有二个不行，

1.对象的同一个synchronized方法是不行的

2.当遇到二个静态的synchronized，是不行的，因为都是类锁，同一个锁。
