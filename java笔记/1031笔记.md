### 4.接口：关键字，interface

研究抽象方法的，

interface Ani{

}

接口含的所有的方法，默认情况都是抽象方法（1.8版本之后，可以含default static方法,要单独声明）

在接口中的方法，abstract 省下，默认也是抽象方法，加上也是

默认就是public abstract

接口如果要含属性，属性就是常量，定值  static final 

接口是不能实例化：有抽象方法

接口也要通过子类来实现(implements) 再重写接口中的方法，来使用的（接口与实现类也是父与子的关系）

多态：继承  重写  子类对象给父类

```
package com.day1031;

//声明一个接口
public interface Ani {
    public static final String title="动物";//常量
    public  void move();
    public abstract void grow();
    public void eat();
}
//一个具体类，来实现接口中的那些抽象方法
class Cat implements Ani {
    @Override
    public void move() {
        System.out.println("猫的移动速度20m/s");
    }
    @Override
    public void grow() {
        System.out.println("猫的成长时间二个月");
    }
    @Override
    public void eat() {
        System.out.println("喜欢吃鱼和面");
    }
}
//
class Dog implements Ani {
    @Override
    public void move() {
        System.out.println("狗的移动速度20m/s");
    }
    @Override
    public void grow() {
        System.out.println("狗的成长时间是一个月");
    }
    @Override
    public void eat() {
        System.out.println("狗喜欢吃骨头");
    }
}
```

接口这种设计的好处：

1 代码的精减和高效

2 用于设计模式

3 多实现（之前单继承时，一个子类对应一个父类，多实现，一个子类可实现多个接口）
