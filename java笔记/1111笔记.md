### 3.集合

数组：相同的类型很多元素组成，定义好数组后，长度是个定值，每次扩展很麻烦

int[] arr=new int[10]

Teacher[] teas=new Teacher[10]

集合就出现了：

1 可变长度的元素的集合，是动态的长度

2 集合提供了很多可用的api,用来操作这些元素

父接口 Collection

- ```
  public interface Collection<E>
  ```

1.不能直接用，要用子类

2.元素值可以为空，也可重复

3.提供很多方法，比如 add

子接口：List

- ```
  public interface List<E>
  extends Collection<E>
  ```

1.不能直接使用

2.提供的元素是有序的，序是指增加时元素时的序，不是数值的值

比如 2 13 3，  不是 2 3 13这个序

3.下标不能重，元素值是可以重

实现类1 ArrayList

- ```
  public class ArrayList<E>
  extends AbstractList<E>
  implements List<E>, RandomAccess, Cloneable, Serializable
  ```

  可调整大小的数组的实现`List`接口

1.内部结果是一个动态数组，每个元素都有下标，通过下标来操作值,连续的

1 String和StringBuffer的区别？StringBuffer和StringBuilder区别？

核心区别主要有两点

1. String长度不能改，就像身份证号一样，改了就会创建新对象

​			StringBuffer就像便签纸一样，可以反复涂改（直接对原对象操作)，长度可变

2. 线程安不安全

​		String天然线程安全

​		StringBuffer自带锁



StringBuffer和StringBuilder的区别

1. Stringbuffer线程安全自带锁，stringbuilder线程不安全

2. Stringbuffer性能慢，StringBuilder性能快

3. 使用场景：StringBuffer多线程并发，

​								Stringbuilder单线程高效操作



2 String是基本数据类型吗？我可不可以写个类继承于String

- String不是基本类型，
- 不能重写String类，因为String类是fianl类不可变

为什么设计成final类

1.安全性：类加载器用到String,防止篡改

2.性能优化：不可变性让字符串常量池成为可能（因为不可变）

3.线程安全：不可变对象天然线程安全
