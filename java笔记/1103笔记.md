### 6.成员内部类

把放类内部的那个类，看成一个成员变量，不须关键字

```
package com.day1103;

public class C2Demo {
    public static void main(String[] args) {
        //先new外部类
        ClassA c1=new ClassA("主教楼403",40);
        //本来Class Ci 看成一个整体，要通过c1.Ci
        //又因为Ci是一个类 又要new Ci,最后返回出来的类型 Ci
       ClassA.Ci c2=  c1.new Ci("java课",6);
       c2.pr();
    }
}
//教室的类--外部类 --outer class
class ClassA{
   String name;
   int num;
   public ClassA(){

   }
   public ClassA(String name,int num){
         this.name=name;
         this.num=num;
   }
   //内部类
   class Ci{
       String namec;
       int sc;//第几节课
       public Ci(){

       }
       public  Ci(String namec,int sc){
           this.namec=namec;
           this.sc=sc;
       }
       //因为Ci是内部类，整个代码块，和外部的属性和方法，是并列的，同一个外部类中，可以相互调用
       public void pr(){
           System.out.println(name+":第"+this.sc+":"+this.namec);
       }
   }

}

```

使用内部类优点：

1.成员内部类是在外部类中的，可以访问外部类中的其它属性和方法

2.内部类也是一种封装，对某些信息的安全性

财务：

员工：（id  name   sal(级别，数字  加薪时间)）

缺点：

1.定义时和调用时，结构和语法比之前稍复杂了些

使用成员内部类：

1.把整个内部类，看成一个属性，当调用时，接收时，又是一个类的类型

2.当类中的某个属性，比较复杂时，要设计成内部类
